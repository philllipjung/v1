# YuniKorn 메트릭 익스포터

**작성일**: 2026년 4월 10일
**버전**: 1.0.0
**목적**: YuniKorn 스케줄러의 애플리케이션 정보를 Prometheus 메트릭으로 노출

---

## 1. 개요

### 1.1 배경

YuniKorn 스케줄러의 기본 메트릭에는 큐별 애플리케이션 상태, 리소스 사용량, 대기 시간 등의 정보가 부족하여, 추가적인 모니터링을 위해 REST API를 호출하여 상세 정보를 수집하는 Prometheus 익스포터를 개발함.

### 1.2 특징

- **단일 메트릭**: `yunikorn_app` 메트릭 하나로 모든 애플리케이션 정보 제공
- **풍부한 레이블**: 큐, 상태, 리소스, 대기 시간 등 7개 레이블 제공
- **실시간 수집**: 15초 간격으로 YuniKorn REST API 호출
- **가벼운 구조**: Go 언어로 개발, 단일 바이너리 (12MB)

## 2. 시스템 구성

### 2.1 아키텍처

```
┌─────────────────┐      ┌──────────────────┐      ┌─────────────────┐
│  Prometheus     │─────▶│  YuniKorn        │─────▶│  YuniKorn       │
│  (Port 9090)    │      │  Exporter        │      │  REST API       │
│                 │      │  (Port 9300)     │      │  (Port 9080)    │
└─────────────────┘      └──────────────────┘      └─────────────────┘
```

### 2.2 컴포넌트

| 컴포넌트 | 포트 | 역할 |
|---------|------|------|
| YuniKorn Scheduler | 9080 | 스케줄링 및 REST API 제공 |
| YuniKorn Exporter | 9300 | 메트릭 수집 및 노출 |
| Prometheus | 9090 | 메트릭 수집 및 저장 |

### 2.3 YuniKorn REST API

**엔드포인트**: `/ws/v1/partition/default/applications/active`

**응답 예시**:
```json
[
  {
    "applicationID": "yunikorn-default-autogen",
    "queueName": "root.@recovery@",
    "applicationState": "Running",
    "usedResource": {
      "vcore": 100,
      "memory": 134217728,
      "pods": 4
    },
    "submissionTime": 1770161351000000000
  }
]
```

## 3. 메트릭

### 3.1 yunikorn_app

**메트릭명**: `yunikorn_app`

**타입**: Gauge

**값**: 항상 1 (레이블을 통해 정보 제공)

**레이블**:

| 레이블 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `queue` | string | 큐 이름 | `root.ias.small`, `root.@recovery@` |
| `state` | string | 애플리케이션 상태 | `Running`, `Accepted`, `New` |
| `app_id` | string | 애플리케이션 ID | `yunikorn-default-autogen` |
| `cpu` | string | 할당된 CPU (밀리코어) | `100`, `500`, `2000` |
| `memory_bytes` | string | 할당된 메모리 (바이트) | `134217728`, `2147483648` |
| `executors` | string | 파드/익스큐서 개수 | `1`, `4`, `10` |
| `wait_seconds` | string | 제출 후 경과 시간 (초) | `5480496`, `3600` |

**메트릭 예시**:
```
yunikorn_app{app_id="yunikorn-default-autogen",cpu="100",executors="4",memory_bytes="134217728",queue="root.@recovery@",state="Running",wait_seconds="5480496"} 1
yunikorn_app{app_id="yunikorn-opensearch-autogen",cpu="500",executors="1",memory_bytes="2147483648",queue="root.@recovery@",state="Running",wait_seconds="4919424"} 1
```

### 3.2 내부 메트릭

| 메트릭명 | 설명 |
|---------|------|
| `yunikorn_exporter_scrape_errors_total` | 스크레이프 에러 총수 (엔드포인트별) |

### 3.3 추적 대상 상태

다음 상태의 애플리케이션을 추적:
- `Running`
- `New`
- `Accepted`
- `Waiting`
- `Submitted`
- `Pending`

## 4. 설치 및 실행

### 4.1 빌드

```bash
cd /root/exporter
go build -o yunikorn-exporter exporter.go
```

### 4.2 실행

**로컬 실행**:
```bash
# YuniKorn 포트 포워딩
kubectl port-forward -n default svc/yunikorn-service 9080:9080

# 익스포터 실행
YUNIKORN_SERVICE_URL=http://localhost:9080 ./yunikorn-exporter
```

**백그라운드 실행**:
```bash
YUNIKORN_SERVICE_URL=http://localhost:9080 nohup ./yunikorn-exporter > exporter.log 2>&1 &
```

### 4.3 환경변수

| 변수 | 기본값 | 설명 |
|-----|--------|------|
| `YUNIKORN_SERVICE_URL` | `http://localhost:9080` | YuniKorn REST API URL |
| `PORT` | `9300` | 메트릭 노출 포트 |
| `SCRAPE_INTERVAL` | `15s` | 스크레이프 간격 |
| `YUNIKORN_PARTITION` | `default` | YuniKorn 파티션 이름 |

## 5. 사용법

### 5.1 메트릭 확인

```bash
# 전체 메트릭
curl http://localhost:9300/metrics

# yunikorn 메트릭만
curl http://localhost:9300/metrics | grep yunikorn_app

# 헬스체크
curl http://localhost:9300/healthz
```

### 5.2 PromQL 쿼리 예시

**특정 큐의 Running 앱**:
```promql
yunikorn_app{queue="root.ias.small", state="Running"}
```

**1시간 이상 대기 중인 앱**:
```promql
yunikorn_app{wait_seconds=~"[3-9][0-9]{3,}|[1-9][0-9]{4,}"}
```

**CPU 1000m (1코어) 이상 사용하는 앱**:
```promql
yunikorn_app{cpu=~"[1-9][0-9]{3,}"}
```

**메모리 1GB 이상 사용하는 앱**:
```promql
yunikorn_app{memory_bytes=~"1[0-9]{9,}|[2-9][0-9]{9,}"}
```

**특정 큐의 대기 시간 합계**:
```promql
sum(yunikorn_app{queue="root.ias.small"} > 0)
```

### 5.3 Grafana 대시보드 예시

**대기 중인 앱 모니터링**:
```promql
# 대기 시간별 앱 수
histogram_quantile(0.95,
  sum(rate(yunikorn_app{state="Accepted"}[5m])) by (wait_seconds)
)
```

**큐별 리소스 사용량**:
```promql
# CPU 사용량 합계
sum by (queue) (
  label_join(yunikorn_app, "cpu_num", "cpu", "cpu") * 0
)
```

## 8. 문제 해결

### 8.1 일반적인 문제

**문제**: `yunikorn_exporter_scrape_errors_total{endpoint="apps"}` 증가

**원인**: YuniKorn REST API 연결 실패

**해결**:
```bash
# 포트 포워딩 확인
kubectl port-forward -n default svc/yunikorn-service 9080:9080

# API 직접 테스트
curl http://localhost:9080/ws/v1/partition/default/applications/active
```

### 8.2 로그 확인

```bash
# 로그 파일 확인
tail -f exporter.log

# 백그라운드 실행 시
journalctl -u yunikorn-exporter -f
```
