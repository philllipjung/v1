# YuniKorn 특정 큐의 Pending 애플리케이션 식별 방법

**테스트 일시**: 2026년 4월 8일
**목적**: YuniKorn 스케줄러에서 특정 큐에 pending/submitted 중인 애플리케이션 이름을 식별하는 방법 검증

---

## 1. 개요

### 1.1 문제 정의

YuniKorn 스케줄러를 사용하는 환경에서 **특정 큐에 pending/submitted 중인 애플리케이션의 이름을 식별**하는 방법을 찾아야 합니다.

### 1.2 테스트 환경

- **Kubernetes Cluster**: minikube (v1.35.0)
- **YuniKorn Scheduler**: v1.7.0 (default 네임스페이스)
- **Spark Operator**: v2.4.0
- **테스트 큐**: root.ias.small, root.default
- **테스트 네임스페이스**: default, ias

## 2. YuniKorn REST API 분석

### 2.1 확인된 엔드포인트

| 엔드포인트 | 제공 정보 | 개별 앱 이름 |
|-----------|----------|--------------|
| `GET /ws/v1/partitions` | 파티션별 앱 카운트 | ❌ |
| `GET /ws/v1/partition/default/queues` | 큐별 리소스 사용량 | ❌ |
| `GET /ws/v1/metrics` | Prometheus 메트릭 | ❌ |

### 2.2 YuniKorn 메트릭 구조

```prometheus

yunikorn_queue_app{queue="root.ias.small",state="submitted"} 0
yunikorn_queue_app{queue="root.ias.small",state="running"} 0
```

**한계**: 개별 애플리케이션 ID/이름을 제공하지 않음

## 5. 구현된 스크립트

### 5.1 find_pending_apps.sh

```bash
#!/bin/bash
# YuniKorn 큐별 Pending 애플리케이션 이름 찾기

TARGET_QUEUE="${1:-}"
NAMESPACE="${2:-default}"

kubectl get sparkapplications -n "$NAMESPACE" -o json | \
  jq -r --arg queue "$TARGET_QUEUE" '
    .items[] |
    select(
      (.status.applicationState.state == "SUBMITTED" or
       .status.applicationState.state == "PENDING" or
       .status.applicationState.state == "RUNNING")
    ) |
    select($queue == "" or .spec.batchSchedulerOptions.queue == $queue) |
    "\(.metadata.name)\t\(.spec.batchSchedulerOptions.queue // "unknown")\t\(.status.applicationState.state)"
  '
```

**사용법**:
```bash
# 특정 큐의 pending 앱
./find_pending_apps.sh "root.ias.small" "default"

# 모든 큐의 pending 앱
./find_pending_apps.sh "" "default"
```

## 6. YuniKorn Placement Rule 설정

### 6.1 기본 구성

```yaml
placement-rules:
  # common 네임스페이스 → root.temp
  - name: common-namespace-rule
    parent: root.temp
    create: false
    filter:
      type: namespace
      values: common

  # default 네임스페이스 → root.default
  - name: default-namespace-rule
    parent: root.default
    create: false
    filter:
      type: namespace
      values: default

  # ias 네임스페이스 → root.ias
  - name: ias-namespace-rule
    parent: root.ias
    create: false
    filter:
      type: namespace
      values: ias
```

### 6.2 주의사항

- **Placement rule 우선**: 네임스페이스에 따라 큐가 자동 할당됨
- **큐 지정 무시**: `batchSchedulerOptions.queue`를 지정해도 placement rule이 우선 적용될 수 있음
- **올바른 네임스페이스 사용**: 원하는 큐를 사용하려면 해당 네임스페이스에서 앱 제출 필요

## 7. 결론

### 7.1 가능 여부

| 방법 | 개별 앱 이름 식별 | 제한 |
|------|------------------|------|
| YuniKorn REST API | ❌ 불가능 | 상태별 카운트만 제공 |
| YuniKorn 메트릭 | ❌ 불가능 | 큐별 카운트만 제공 |
| Kubernetes API | ✅ 가능 | SparkApplication/Pod 조회 |

### 7.2 최종 솔루션

**Kubernetes API를 사용하여 특정 큐의 pending/submitted 애플리케이션 이름 식별 가능**

```bash
# 완성된 쿼리
kubectl get sparkapplications -n <namespace> -o json | \
  jq '.items[] |
      select(.spec.batchSchedulerOptions.queue == "<queue-name>") |
      select(.status.applicationState.state == "SUBMITTED" or
              .status.applicationState.state == "PENDING") |
      {name: .metadata.name,
       queue: .spec.batchSchedulerOptions.queue,
       state: .status.applicationState.state,
       appId: .status.sparkApplicationId}'
```

### 7.3 확인된 상태 값

- `NEW`: 애플리케이션이 생성됨
- `SUBMITTED`: YuniKorn에 제출됨 (pending 상태)
- `ACCEPTED`: 큐에서 수락됨
- `RUNNING`: 실행 중
- `COMPLETED`: 완료됨
- `FAILED`: 실패함
- `REJECTED`: 거부됨
