# YuniKorn REST API 테스트 문서

**테스트 일시**: 2026년 4월 9일
**목적**: YuniKorn 스케줄러 REST API를 통해 큐 및 애플리케이션 정보를 조회하는 방법 검증

---

## 1. 개요

### 1.1 테스트 목표

YuniKorn 스케줄러 REST API를 통해 다음 정보를 조회하는 방법 검증:

1. **큐 정보 조회**: 특정 큐의 리소스 사용량 및 설정 확인
2. **애플리케이션 조회**: 특정 큐/상태별 애플리케이션 목록 확인
3. **애플리케이션 상세**: 개별 애플리케이션의 상태 및 리소스 사용량 확인

### 1.2 테스트 환경

- **Kubernetes Cluster**: minikube (v1.35.0)
- **YuniKorn Scheduler**: v1.7.0 (apache/yunikorn:scheduler-1.7.0)
- **YuniKorn Web Service**: Port 9080
- **테스트 큐**: root.ias.small, root.@recovery@
- **테스트 네임스페이스**: default

---

## 2. YuniKorn REST API 엔드포인트

### 2.1 발견된 API 엔드포인트

YuniKorn 소스 코드(`pkg/webservice/routes.go`)에서 확인된 주요 엔드포인트:

| 엔드포인트 | 설명 | 동작 여부 |
|-----------|------|----------|
| `GET /ws/v1/partitions` | 전체 파티션 정보 | ✅ |
| `GET /ws/v1/partition/:partition/queues` | 큐 계층 구조 | ✅ |
| `GET /ws/v1/partition/:partition/queue/:queue` | 특정 큐 정보 | ✅ |
| `GET /ws/v1/partition/:partition/queue/:queue/applications` | 특정 큐의 모든 앱 | ✅ |
| `GET /ws/v1/partition/:partition/applications/:state` | 상태별 애플리케이션 | ✅ |
| `GET /ws/v1/partition/:partition/application/:appid` | 특정 앱 상세 정보 | ✅ |
| `GET /ws/v1/metrics` | Prometheus 메트릭 | ✅ |

### 2.2 API 상태 필터

지원되는 상태 값:
- `active`: 실행 중 및 수락된 애플리케이션
- `rejected`: 거부된 애플리케이션
- `completed`: 완료된 애플리케이션

## 3. YuniKorn 구성

### 3.1 현재 큐 구조

**큐 계층도:**
```
root
├── root.default
├── root.opensearch
├── root.ias
│   ├── root.ias.small (max: 10GB memory, 2000m vcore)
│   ├── root.ias.medium (max: 20GB memory, 4000m vcore)
│   └── root.ias.large (max: 40GB memory, 8000m vcore)
├── root.temp
└── root.@recovery@ (자동 복구 큐)
```

### 3.2 Placement Rules

```yaml
placement-rules:
  # common 네임스페이스 → root.temp
  - name: common-namespace-rule
    parent: root.temp
    filter:
      type: namespace
      values: common

  # default 네임스페이스 → root.default
  - name: default-namespace-rule
    parent: root.default
    filter:
      type: namespace
      values: default

  # ias 네임스페이스 → root.ias
  - name: ias-namespace-rule
    parent: root.ias
    filter:
      type: namespace
      values: ias
```

### 3.3 큐 리소스 설정

| 큐 | Max Memory | Max VCore | Max Apps | Max Running |
|----|-----------|-----------|----------|--------------|
| root.ias.small | 10GB | 2000m | 10 | 5 |
| root.ias.medium | 20GB | 4000m | 20 | 10 |
| root.ias.large | 40GB | 8000m | 5 | 3 |

## 7. 제한 사항 및 주의사항

### 7.1 상태 필터링 제한

- `active`: New, Accepted, Running 상태 포함
- `rejected`: 거부된 앱만 포함
- `completed`: 완료된 앱만 포함
- `pending` 또는 `submitted`: 별도 상태로 제공되지 않음 (active에 포함)

### 7.2 API 접근 방법

```bash
# Port-forward 설정
kubectl port-forward -n default svc/yunikorn-service 9080:9080

# API 호출
curl http://localhost:9080/ws/v1/partition/default/applications/active
```

### 7.3 큐 경로 포맷

- API 경로: `/ws/v1/partition/default/queue/root.ias.small/applications`
- 큐 경로: `root.ias.small` (점 표기)
- 주의: 특수문자(`@`)가 포함된 큐는 URL 인코딩 필요
