# SparkApplication Exporter Design Document

**Date**: 2026-04-29
**Author**: Spark Operator Metrics Team
**Version**: 1.0

## 1. Overview

SparkApplication Exporter는 Kubernetes에서 실행되는 SparkApplication CR(Custom Resource)을 모니터링하여, 커스텀 라벨을 포함한 메트릭을 Prometheus에 노출하는 오픈소스 프로젝트입니다.

## 2. Requirements

### 2.1 Functional Requirements

1. **SparkApplication CR 감시**
   - Kubernetes API를 통해 SparkApplication CR의 생성, 수정, 삭제 이벤트 감시
   - 여러 네임스페이스 동시 감시 지원

2. **메트릭 수집 및 노출**
   - 스파크 잡의 총 처리 시간 (초)
   - 펜딩 상태의 총 기간 (초)
   - 스파크 잡 상태 (완료/실패)
   - 어플리케이션별 처리 개수

3. **커스텀 라벨 지원**
   - SparkApplication CR의 metadata.labels에서 커스텀 라벨 추출
   - 모든 메트릭에 커스텀 라벨 포함

### 2.2 Non-Functional Requirements

1. **성능**
   - 메트릭 수집 주기: 30초 (설정 가능)
   - 메트릭 노출 latency: < 100ms

2. **안정성**
   - Kubernetes API 연결 실패 시 재시도
   - 메트릭 수집 실패 시 에러 카운트

3. **확장성**
   - 여러 네임스페이스 감시
   - 동시에 수백 개의 SparkApplication 처리

## 3. Metrics Specification

### 3.1 spark_app_processing_time_seconds

**타입**: Gauge
**설명**: 스파크 잡의 총 처리 시간 (초)

**라벨**:
```yaml
app_name:      애플리케이션 이름 (예: clustering)
app_type:      애플리케이션 타입 (예: ias)
provision_id:  프로비저닝 ID (예: 0002_wfbm)
queue:         YuniKorn 큐 (예: root.ias.small)
service_id:    서비스 ID (예: test-spark-app)
service_name:  서비스 이름 (예: analytics)
namespace:     Kubernetes 네임스페이스
```

**계산公式**:
```
processing_time = termination_time - submission_time
```

**예시**:
```
spark_app_processing_time_seconds{
  app_name="clustering",
  app_type="ias",
  provision_id="0002_wfbm",
  queue="root.ias.small",
  service_id="test-spark-app",
  service_name="analytics",
  namespace="default"
} 45.2
```

### 3.2 spark_app_pending_duration_seconds

**타입**: Gauge
**설명**: 펜딩 상태의 총 기간 (초)

**라벨**: spark_app_processing_time_seconds와 동일

**계산公式**:
```
pending_duration = running_time - submitted_time
```

**참고**: 현재 구현에서는 placeholder 값 (2.0초) 사용. 실제 state transition 추적 필요.

### 3.3 spark_app_status

**타입**: Gauge
**설명**: 스파크 잡 상태

**라벨**: spark_app_processing_time_seconds + state
```
state: COMPLETED, FAILED, RUNNING, PENDING, SUBMITTED
```

**값**:
```
1.0 = COMPLETED (성공)
0.0 = FAILED (실패)
-1.0 = RUNNING (실행 중)
```

**예시**:
```
spark_app_status{
  app_name="clustering",
  app_type="ias",
  ...,
  state="COMPLETED"
} 1.0
```

### 3.4 spark_app_count_total

**타입**: Counter
**설명**: 처리된 어플리케이션 총 개수

**라벨**: spark_app_processing_time_seconds + status
```
status: COMPLETED, FAILED
```

**예시**:
```
spark_app_count_total{
  app_name="clustering",
  app_type="ias",
  ...,
  status="COMPLETED"
} 15
```

### 3.5 spark_app_resource_cores_total

**타입**: Gauge
**설명**: 요청된 총 CPU 코어 수 (Driver + Executors)

**라벨**: spark_app_processing_time_seconds + component
```
component: driver, executors, total
```

**계산공식**:
```
total_cores = driver.cores + (executor.cores × executor.instances)
```

**예시**:
```
spark_app_resource_cores_total{
  app_name="clustering",
  app_type="ias",
  ...,
  component="driver"
} 1

spark_app_resource_cores_total{
  app_name="clustering",
  app_type="ias",
  ...,
  component="executors"
} 1

spark_app_resource_cores_total{
  app_name="clustering",
  app_type="ias",
  ...,
  component="total"
} 2
```

**데이터 출처**: `spec.driver.cores`, `spec.executor.cores`, `spec.executor.instances`

### 3.6 spark_app_resource_memory_bytes_total

**타입**: Gauge
**설명**: 요청된 총 메모리 (Bytes) (Driver + Executors)

**라벨**: spark_app_processing_time_seconds + component
```
component: driver, executors, total
```

**계산공식**:
```
total_memory = driver.memory + (executor.memory × executor.instances)
```

**메모리 포맷 변환**:
```
2048m → 2048 * 1024 * 1024 = 2147483648 bytes
1Gi → 1024^3 = 1073741824 bytes
```

**예시**:
```
spark_app_resource_memory_bytes_total{
  app_name="clustering",
  app_type="ias",
  ...,
  component="driver"
} 2.147483648e+09  # 2GB

spark_app_resource_memory_bytes_total{
  app_name="clustering",
  app_type="ias",
  ...,
  component="executors"
} 2.147483648e+09  # 2GB

spark_app_resource_memory_bytes_total{
  app_name="clustering",
  app_type="ias",
  ...,
  component="total"
} 4.294967296e+09  # 4GB
```

**데이터 출처**: `spec.driver.memory`, `spec.executor.memory`, `spec.executor.instances`

### 3.7 spark_app_executor_count

**타입**: Gauge
**설명**: 요청된 Executor 개수

**라벨**: spark_app_processing_time_seconds (component 없음)

**예시**:
```
spark_app_executor_count{
  app_name="clustering",
  app_type="ias",
  ...,
  namespace="default"
} 1
```

**데이터 출처**: `spec.executor.instances`

### 3.8 spark_app_failure_count_total

**타입**: Counter
**설명**: 실패한 Spark Application 총 개수

**라벨**: spark_app_processing_time_seconds (status 없음)

**증가 조건**: `applicationState.state == "FAILED"`

**예시**:
```
spark_app_failure_count_total{
  app_name="clustering",
  app_type="ias",
  ...,
  namespace="default"
} 3
```

**데이터 출처**: `status.applicationState.state`, `status.executionAttempts`

### 4.2 Component Description

#### 4.2.1 CR Watcher (Dynamic Informer)
- Kubernetes Dynamic Client 사용
- SparkApplication CRD 감시
- Real-time 이벤트 수신 (Add/Modify/Delete)

#### 4.2.2 Event Handler
- CR 이벤트 처리
- 커스텀 라벨 추출
- 메트릭 계산

#### 4.2.3 Metrics Store
- In-memory 메트릭 저장
- 최신 상태 유지
- CR 삭제 시 정리

#### 4.2.4 Prometheus Registry
- Prometheus client_golang 사용
- 표준 Prometheus 형식으로 노출

## 5. Data Structures

### 5.1 SparkApplication CR Structure

```go
type SparkApplication struct {
    Metadata metav1.ObjectMeta `json:"metadata"`
    Status   SparkAppStatus    `json:"status"`
}

type SparkAppStatus struct {
    ApplicationState          AppState `json:"applicationState"`
    LastSubmissionAttemptTime string   `json:"lastSubmissionAttemptTime"`
    TerminationTime           string   `json:"terminationTime"`
    SubmissionAttempts        int      `json:"submissionAttempts"`
    ExecutionAttempts         int      `json:"executionAttempts"`
}

type AppState struct {
    State        string `json:"state"`  // SUBMITTED, PENDING, RUNNING, COMPLETED, FAILED
    ErrorMessage string `json:"errorMessage"`
}
```

### 5.2 Custom Labels Structure

```go
type CustomLabels struct {
    AppName     string // metadata.labels["app-name"]
    AppType     string // metadata.labels["app_type"]
    ProvisionID string // metadata.labels["provision_id"]
    Queue       string // metadata.labels["queue"]
    ServiceID   string // metadata.labels["service-id"]
    ServiceName string // metadata.labels["service-name"]
    Namespace   string // metadata.namespace
}
```

### 5.3 Metrics Data Structure

```go
type MetricsData struct {

    Status          string

### 6.1 Kubernetes Client Setup

```go
// In-cluster config
config, err := rest.InClusterConfig()
if err != nil {
    // Fallback to kubeconfig
    config, err = clientcmd.BuildConfigFromFlags("", kubeconfig)

// Dynamic client for CRDs
dynamicClient, err := dynamic.NewForConfig(config)

### 6.2 CR Watcher Setup

```go
sparkAppGVR := schema.GroupVersionResource{
    Group:    "sparkoperator.k8s.io",
    Version:  "v1beta2",
    Resource: "sparkapplications",
}

watcher, err := dynamicClient.
    Resource(sparkAppGVR).
    Namespace(namespace).
    Watch(ctx, metav1.ListOptions{})
```

### 6.3 Event Processing

```go
for event := range watcher.ResultChan() {
    obj := event.Object.(*unstructured.Unstructured)

    switch event.Type {
    case watch.Added, watch.Modified:
        updateMetrics(obj)
    case watch.Deleted:
        deleteMetrics(obj)
    }
}
```

### 6.4 Metric Calculation

```go
func calculateProcessingTime(submissionTime, terminationTime string) float64 {
    submission, _ := time.Parse(time.RFC3339Nano, submissionTime)
    termination, _ := time.Parse(time.RFC3339Nano, terminationTime)
    return termination.Sub(submission).Seconds()
}

func isTerminalState(state string) bool {
    return state == "COMPLETED" || state == "FAILED"
}
```

### 7.5 ServiceMonitor

The ServiceMonitor defines how Prometheus scrapes the exporter metrics.

```yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: spark-app-exporter
  namespace: default
  labels:
    app: spark-app-exporter
spec:
  selector:
    matchLabels:
      app: spark-app-exporter
  endpoints:
  - port: metrics
    interval: 30s
    path: /metrics
```

**Key Points**:
- **selector.matchLabels**: Must match the Service labels
- **endpoints.port**: Must match Service port name
- **interval**: How often Prometheus scrapes metrics
- **path**: Metrics endpoint path (default: /metrics)

## 8. Prometheus Timestamps and Time Series

### 8.1 How Prometheus Adds Timestamps

**Important**: The exporter output format does NOT include explicit timestamps:

```
# Exporter output (no timestamp):
spark_app_processing_time_seconds{...} 32
```

However, **Prometheus automatically adds timestamps** when scraping:

```
# Prometheus storage (with timestamp):
spark_app_processing_time_seconds{...} 32 1777509747.466
                                                    ^^^^^^^^^^^^^^^^
                                                    Prometheus adds this
```

### 8.2 Time Series Data Flow

```
┌─────────────────┐     ┌──────────────┐     ┌─────────────┐
│ Spark Exporter  │────▶│ Prometheus   │────▶│ Time Series │
│                 │     │ Operator     │     │ Database    │
│ Output:         │     │              │     │             │
│ metric value    │     │ Scrapes      │     │ Value +     │
│                 │     │ every 30s    │     │ Timestamp   │
└─────────────────┘     └──────────────┘     └─────────────┘
```

### 8.3 Prometheus Query API Response

When you query Prometheus, each data point includes a timestamp:

```json
{
  "metric": {
    "__name__": "spark_app_processing_time_seconds",
    "app_name": "clustering",
    "service_name": "analytics"
  },
  "value": [
    1777509747.466,  ← Timestamp (Unix epoch)
    "32"             ← Metric value
  ]
}
```

### 8.4 Time Series Chart Use Cases

| Use Case | Works? | Notes |
|----------|--------|-------|
| Track job completion times over time | ✅ Yes | Each scrape creates a time point |
| Compare apps by processing time | ✅ Yes | Group by labels |
| Real-time job progress | ❌ No | Value only appears after job completes |
| Historical trends | ✅ Yes | Prometheus stores time-series data |
| Alerting on slow jobs | ✅ Yes | Threshold-based alerts |

```

### 11.1 No Metrics Appearing

**증상**: `/metrics` 엔드포인트에 메트릭이 없음

**원인**:
- SparkApplication CR이 없음
- 네임스페이스 설정 오류
- RBAC 권한 부족

**해결**:
```bash
# CR 확인
kubectl get sparkapplications -A

# 네임스페이스 확인
kubectl get pods -n default

# 로그 확인
kubectl logs -f deployment/spark-app-exporter
```

### 11.2 Labels Missing

**증상**: 메트릭에 커스텀 라벨이 "unknown"으로 표시

**원인**:
1. **오래된 SparkApplication**: 커스텀 라벨이 없는 이전 버전 CR이 존재
2. **라벨 누락**: SparkApplication CR에 라벨이 정의되지 않음
3. **라벨 키 불일치**: 템플릿과 CR의 라벨 키가 다름

**해결**:

**방법 1: 오래된 CR 삭제**
```bash
# 확인
kubectl get sparkapplications -A

# unknown 라벨이 있는 CR 삭제
kubectl delete sparkapplication regression-analysis -n default
```

**방법 2: 라벨 추가**
```bash
# 커스텀 라벨 추가
kubectl label sparkapplication <name> \
  app-name=regression \
  app_type=ias \
  service-name=regression \
  service-id=regression-analysis \
  provision_id=manual
```

**방법 3: 라벨 확인**
```bash
# CR의 전체 라벨 확인
kubectl get sparkapplication <name> -o yaml | grep labels: -A 10

# 필요한 라벨이 있는지 확인
kubectl get sparkapplication <name> -o jsonpath='{.metadata.labels}' | jq 'with_entries(select(.key | test("app-|service-|provision_id|queue")))'
```

# 템플릿 확인
grep "app-name:" /root/hynix/template/*.yaml
```

### 11.3 Connection Refused

**증상**: Kubernetes API 연결 실패

**원인**:
- ServiceAccount缺少
- ClusterRoleBinding缺少

**해결**:
```bash
# RBAC 확인
kubectl get serviceaccount spark-app-exporter
kubectl get clusterrolebinding spark-app-exporter

# 재생성
kubectl apply -f rbac.yaml
